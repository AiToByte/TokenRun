### 第一部分：系统集成拓扑 (System Integration Map)

在代码实现上，`TokenRun` 的启动类将作为整个炼金厂的“主控台”。

```python
# tokenrun/main.py (伪代码实现逻辑)

class TokenRunApp:
    def __init__(self, runfile_path: str):
        # 1. 初始化核心组件
        self.runfile = Runfile.load(runfile_path)
        self.gateway = ResourceGateway(self.runfile.security)
        self.ledger = TokenLedger(budget_usd=self.runfile.governance.max_usd)
        self.vault = SkillVault(storage_path="./vault")
        self.telemetry = TelemetryManager()
        
        # 2. 注入 Runner
        self.runner = TRRunner(
            runfile=self.runfile,
            gateway=self.gateway,
            ledger=self.ledger,
            telemetry=self.telemetry
        )

    async def run_mission(self):
        # 第一阶段：1% 采样与打样
        print("--- [PHASE 1: SAMPLING] ---")
        sample_report = await self.runner.execute(mode="sample")
        
        # 触发人工干预点 (Human-in-the-Loop)
        if self.runfile.sampling.auto_pause:
            user_decision = await self.telemetry.wait_for_approval(sample_report)
            if user_decision.action == "EDIT":
                # 记录版本变更，重新执行采样
                self.runfile.update_prompt(user_decision.new_prompt)
                return await self.run_mission() 

        # 第二阶段：全量生产
        print("--- [PHASE 2: MASS PRODUCTION] ---")
        final_trace = await self.runner.execute(mode="full")
        
        # 第三阶段：资产化固化
        print("--- [PHASE 3: SOLIDIFICATION] ---")
        skill_id = await self.vault.solidify(final_trace, self.runfile)
        print(f"🎉 任务圆满完成！资产已存入技能库: {skill_id}")
```

### 第二部分：MVP 验证场景 —— “私人财务深度审计”

这是一个能够充分体现 TokenRun 价值的场景：

*   **痛点：** 银行导出的账单极其混乱，包含大量隐私（姓名、账号、具体交易对手），且简单的分类器容易出错。
*   **TokenRun 解决方案：**
    1.  **脱敏：** 确保银行卡号和姓名在发送给云端前被本地 Gateway 屏蔽。
    2.  **Loop 修正：** 如果 AI 误将“购买云服务器”归类为“娱乐消费”，Critic（廉价模型）根据规则将其打回，让 Actor 重新分类并提取具体的业务意义。
    3.  **资产化：** 跑完后，你会获得一个针对你个人消费习惯定制的“财务分类 Skill”，下次使用确定性 100%。

#### 该场景下的 MVP Runfile 定义：

```yaml
name: "Personal_Finance_Refinery"
```

### 第三部分：落地执行的四个关键技术细节

#### 1. 确定性指纹的“版本链”
当用户在“采样阶段”点击 `Edit & Resample` 时，Runner 内部会生成如下记录：
*   `v1.0 (Initial)` -> Passed: 40% -> Cost: $0.01
*   `v1.1 (Modified)` -> Passed: 95% -> Cost: $0.012
*   **系统动作：** 自动将 `v1.1` 的模板哈希写入 `fingerprint.locked_prompt_hash`。

#### 2. 本地网关的“双向还原”
为了保证输出有用，Gateway 需要具备：
*   **发送前：** `Zhang San` $\rightarrow$ `[NAME_1]`。
*   **接收后：** 如果输出中包含 `[NAME_1]`，本地 Runner 根据映射表还原回 `Zhang San`（云端始终不知道真实姓名）。

#### 3. Critic 的“多维评分量表”
在执行 `LLM-Critic` 评估时，我们不只让它返回 `Yes/No`，而是返回一个评分向量：
*   `accuracy`: 0.9, `privacy`: 1.0, `consistency`: 0.8。
*   **逻辑：** 只有加权总分 > 0.85 且 `privacy` 必须为 1.0 时，Loop 才会退出。

#### 4. 账本的“ROI 动态计算”
任务结束时，Ledger 会输出：
> “本次任务消耗 Token 250万，费用 $12.5。产出精炼数据 500 条。相比人工处理，效率提升 400x。已固化为 Permanent Skill，下次运行成本预计降低 15%（因逻辑已锁定，重试率将降低）。”
