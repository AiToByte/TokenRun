### 1. 项目基础定义：`Runfile` 蓝图协议

我们将 `Runfile` 定义为一个高度结构化的声明式模型。

```python
from enum import Enum
from typing import List, Optional, Dict, Any, Union
from pydantic import BaseModel, Field, HttpUrl

class DeterminismLevel(str, Enum):
    STRICT = "strict"      # 锁定一切参数，不一致则报错
    FLEXIBLE = "flexible"  # 允许微小偏差或模型小版本升级

class LoopStrategy(str, Enum):
    FEEDBACK_DRIVEN = "feedback-driven" # 基于 Critic 反馈修正
    EXHAUSTIVE = "exhaustive"           # 跑满最大次数取最优
    ONCE = "once"                       # 仅执行一次

class ResourceType(str, Enum):
    LOCAL_FILE = "local_file"
    SQL_QUERY = "sql_query"
    S3_OBJECT = "s3_object"
    API_ENDPOINT = "api_endpoint"

# --- 资源与安全定义 ---
class Resource(BaseModel):
    id: str
    uri: str
    type: ResourceType
    description: Optional[str] = None

class SecurityConfig(BaseModel):
    masking_rules: List[str] = Field(default_factory=lambda: ["emails", "api_keys"])
    local_sandbox: bool = True

# --- 采样与确定性逻辑 ---
class SamplingConfig(BaseModel):
    enabled: bool = True
    mode: str = "percentage"  # percentage | count
    value: float = 0.01       # 1%
    auto_pause: bool = True   # 采样后是否自动暂停等待确认

class Fingerprint(BaseModel):
    model_id: str
    prompt_hash: str
    parameters: Dict[str, Any]
    snapshot: Optional[str] = None # 采样成功时的样本快照哈希

# --- 循环工程定义 (Loop Engineering) ---
class ValidationRule(BaseModel):
    type: str # regex | json_schema | llm_eval | code_eval
    criteria: Any
    weight: float = 1.0

class LoopConfig(BaseModel):
    strategy: LoopStrategy = LoopStrategy.FEEDBACK_DRIVEN
    max_attempts: int = 3
    exit_criteria: List[ValidationRule]
    retry_delay: int = 1 # 秒

# --- 任务节点与工作流 ---
class TaskNode(BaseModel):
    id: str
    name: str
    depends_on: List[str] = []
    actor_prompt_template: str
    loop_config: LoopConfig
    config: Dict[str, Any] = {}

class Runfile(BaseModel):
    version: str = "1.0"
    metadata: Dict[str, str]
    context: List[Resource]
    security: SecurityConfig
    sampling: SamplingConfig
    workflow: List[TaskNode]
    fingerprint: Optional[Fingerprint] = None
```

### 2. 执行状态定义：`Trace` 与 `Packet`

为了实现“实时查看”和“断点续传”，我们需要定义每一个原子步骤在循环中的执行状态。

```python
class EvaluationResult(BaseModel):
    passed: bool
    score: float
    critique: Optional[str] = None # 评估器的反馈意见
    suggestions: List[str] = []    # 针对下一轮循环的改进建议

class ExecutionIteration(BaseModel):
    iteration_index: int
    input_payload: str
    output_content: str
    evaluation: EvaluationResult
    tokens_consumed: Dict[str, int]
    latency_ms: int
    timestamp: str

class TaskTrace(BaseModel):
    task_id: str
    status: str # pending | running | paused | completed | failed
    iterations: List[ExecutionIteration]
    final_output: Optional[str] = None
```

### 3. 高屋建瓴的设计考量

1.  **解耦意图与实现：** `TaskNode` 中只记录 `actor_prompt_template`，而不写死 Skill 的具体代码。Runner 会在运行时根据 `id` 自动匹配本地的 Skill 逻辑。
2.  **强制验证：** `LoopConfig` 中必须包含 `exit_criteria`。这强制开发者在写任务时就必须思考：“什么样才算做好了？”——这是**工业化 Agent** 的核心。
3.  **确定性存证：** `Fingerprint` 包含 `prompt_hash`。如果用户修改了 Prompt，指纹将失效，强制触发重新采样。
4.  **Token 审计原生化：** 每个 `ExecutionIteration` 都自带 `tokens_consumed`。这使得我们可以在任务运行到任何一个点时，都能生成一份极其精确的账单报告。
