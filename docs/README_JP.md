## 概要

TokenRun は、**ループエンジニアリング**を通じて信頼性の低い AI 出力を工業品質の結果に変換する、生産グレードのフレームワークです。

コアメカニズム：**Actor-Critic ループ**

1. **Actor**（高価なモデル、例：GPT-4o）がタスクを実行し出力を生成
2. **Critic**（安価なモデル、例：GPT-4o-mini）が品質を監査しフィードバックを提供
3. 品質が基準に達しない場合、フィードバックを注入して Actor が再試行
4. 品質基準を満たすか予算が尽きるまでループが継続

## 主な特徴

### 実行エンジン

| 特徴 | 説明 |
|---|---|
| **ループエンジニアリング** | Actor-Critic フィードバックループ、3つの戦略：フィードバック駆動、徹底的、単発 |
| **プログラム的検証** | `regex` と `json_schema` ルールは LLM 呼び出しなしで検証 |
| **多次元スコアリング** | Critic が次元別スコア（正確性、完全性、形式）を返却 |
| **動的モデルルーティング** | N回失敗後に安価なモデルから高価なモデルへ自動エスカレーション |

### 安全性と確定性

| 特徴 | 説明 |
|---|---|
| **プライバシー脱感作** | 可逆 PII マスキング（メール、電話、身分証、IP、API Key） |
| **予算サーキットブレーカー** | リアルタイム USD トラッキング、予算超過で自動停止 |
| **フィンガープリントロック** | モデルID + プロンプトハッシュ + temperature + seed をロック |

### 資産とエコシステム

| 特徴 | 説明 |
|---|---|
| **スキル固化** | 最適プロンプト + ゴールデンサンプルを `.trs` スキルパッケージに抽出 |
| **スキルチェーン** | Runfile ノードで `.trs` ファイルを参照 |
| **知識蒸留** | [入力]→[出力] ペアをファインチューニングデータセットとしてエクスポート |
